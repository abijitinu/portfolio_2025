A web portal to create, maintain, analyze, and optimize marketing campaigns.

The Context

P&G, a global consumer goods corporation, manages thousands of marketing campaigns across various regions and brands. They needed a centralized platform to streamline the creation, maintenance, and analysis of these campaigns to ensure consistency and efficiency.

The Brief

The goal was to design a comprehensive web portal that allows users to create campaigns, manage operations, analyze performance, and gather insights.

Key Requirements

  • B2B Application with multiple user roles
  • Robust Identity Access Management (IAM)
  • Custom Report Generation & Analytics
  • Flagging, error reporting, and helpdesk integration

The Audience

The primary users of this system are Brand Managers, Regional Marketing Leads, and System Administrators. They need a tool that is powerful yet intuitive enough to manage complex data without requiring deep technical knowledge.

The Process

We adopted an iterative design process, starting with understanding the complex hierarchy of P&G's brand ecosystem. Through stakeholder interviews and workshops, we mapped out the user journeys for campaign creation and approval.

Wireframing focused on simplifying the navigation and ensuring that critical data was always accessible. Regular feedback loops with the development team ensured technical feasibility.

The Solution

Homepage

The homepage provides a high-level overview of client engagements. Users can see a glimpse of the brands being handled, the number of active and delivered campaigns, country exposure, and total opt-in statistics.

Dashboard

The dashboard offers multi-level filtering to drill down into specific campaigns. It presents comprehensive data including:

  • Total and Unique Visitors
  • Total Registrations (Opt-in users)
  • Survey metrics related to campaigns

IAM Module

For a multi-locale company, managing access across different levels is critical. The Identity Access Management (IAM) module allows for granular control:

  • Role-Based Filtering: Select brands, campaigns, and specific user roles.
  • Privilege Management: Assign specific privileges to users.
  • Request Workflow: Users can request access for themselves or others with comments. Admins can review, accept, or reject requests with feedback.

Custom Reports

Reports are highly customizable to handle data from multiple brands, campaigns, and locales. Since a single campaign can run across different countries, the system allows for generating separate data reports for each locale as well as combined global reports.

The Outcome

The new Campaign System significantly reduced the time required to launch and manage campaigns. The unified dashboard provided stakeholders with real-time visibility into performance, enabling faster data-driven decisions. The robust IAM system ensured data security and proper access control across the global organization.

Lessons Learned

Designing for a B2B enterprise environment taught me the importance of scalability and flexibility. Balancing granular control with ease of use was a key challenge that we overcame through rigorous user testing and iteration.
